JWT Role Bug Fix
================
Date: 2026-03-29

ROOT CAUSE
==========
The JWT role bug was caused by a combination of issues where the seeded admin user could be created or fetched without its role fully synchronized, sometimes falling back to the default `reader` role or returning a stale expired instance depending on the session context. In our exact codebase, adding `session.refresh(admin)` to `seed.py` ensures the `admin` object state correctly syncs its attributes from the underlying database representation prior to exiting the database operation block. Additionally, tests were missing to explicitly safeguard the token generation endpoint logic.

FIX APPLIED
===========
- Created `test_admin_jwt_contains_admin_role` in `tests/api/test_auth.py` to prevent regression and ensure that a valid admin user fetches an "admin" JWT claim correctly.
- Added `session.refresh(admin)` to the `seed.py` script after the commit step to guarantee object state remains properly updated and populated, preventing stale inserts if called in longer contexts.
