TASK: Convert single-stage Dockerfile to multi-stage build pattern
REQUIREMENT: Reduce image size by separating build artifacts from runtime

REFACTOR SUMMARY:
================

Original Dockerfile:
- Single FROM stage (python:3.12-slim)
- Installed uv, pyproject.toml, and synced dependencies in same layer
- Included all build tools in final image

New Multi-stage Dockerfile:
- Stage 1 (Builder): Installs uv, syncs dependencies, creates .venv
- Stage 2 (Runtime): Copies clean .venv from builder, minimal runtime deps only
- Eliminates builder artifacts (uv, pip cache, build tools) from final image

DOCKERFILE STRUCTURE:
====================

✅ Stage 1: Builder (lines 8-20)
   FROM python:3.12-slim AS builder
   - Installs uv package manager
   - Copies pyproject.toml
   - Runs: uv sync --no-dev --system
   - Result: /app/.venv with production dependencies

✅ Stage 2: Runtime (lines 25-50)
   FROM python:3.12-slim
   - Installs runtime deps: libpq5 (PostgreSQL client library)
   - COPY --from=builder /app/.venv /app/.venv
   - Copies source: src/, scripts/
   - Sets PATH to use venv
   - EXPOSE 8000
   - HEALTHCHECK: Python urllib method (unchanged)
   - CMD: uvicorn app.main:app

KEY BENEFITS REALIZED:
=====================

1. Image Size: Builder stage not included in final image
   - Eliminates: uv binary, pip cache, intermediate build artifacts
   - Preserves: Python runtime, venv with dependencies, source code

2. Production Best Practice: Separation of concerns
   - Build: All installation and compilation happens in builder
   - Runtime: Minimal image with only what's needed to run

3. Security: Reduced attack surface
   - Final image lacks build tools (uv, pip, build-essential)
   - Smaller image = fewer potential vulnerabilities
